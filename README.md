# تطبيق إدارة الهوتسبوت / يوزر مانجر — نسخة الويب

تطبيق ويب لإدارة راوتر MikroTik (هوتسبوت + يوزر مانجر): مراقبة الموارد، توليد كروت الدخول
وطباعتها، أدوات شبكة (Ping، مراقبة الترافيك، الأجهزة المجاورة)، وإعدادات الربط والقوالب.
مبني حسب خارطة الطريق الواردة في التحليل: واجهة أمامية + Backend بـ PHP يتحدث مباشرة
مع **MikroTik RouterOS API** (منفذ 8728 / 8729) + قاعدة بيانات MySQL محلية للأرشفة.

## كيف تتصل بالراوتر الحقيقي (مثل Winbox)

الواجهة الآن تبدأ بشاشة دخول: عنوان الراوتر، المنفذ، اسم المستخدم، كلمة المرور —
تماماً كشاشة Winbox. عند الضغط على "اتصال" تُرسل البيانات إلى `backend/api/connect.php`
الذي يتصل فعلياً بالراوتر، يتحقق من البيانات، ويحفظ الجلسة في PHP Session. بعدها كل
الشاشات (الموارد، الكروت، Ping، الترافيك، الجيران، السجل، IP Pools) تسحب بيانات حقيقية.

**نقطة مهمة جداً:** هذا Backend لا بد أن يعمل على جهاز موجود **على نفس الشبكة المحلية**
للراوتر (أو يصل إليه عبر VPN/منفذ موجَّه)، لأن متصفح الجوال لا يمكنه فتح اتصال TCP خام
ببروتوكول RouterOS API مباشرة — هذا هو سبب وجود الـ Backend بالأصل.

### تشغيل سريع للتجربة من جهازك مباشرة (بدون أي استضافة)

```bash
cd connect-visual          # المجلد الذي يحوي frontend/ و backend/
php -S 0.0.0.0:8000        # خادم PHP المدمج، يكفي لأي اختبار محلي
```

ثم من متصفح جوالك (وهو على نفس شبكة الواي فاي التي عليها الراوتر والكمبيوتر):

```
http://عنوان-جهازك-المحلي:8000/frontend/index.html
```

أدخل عنوان الراوتر (مثل `192.168.88.1`)، اسم المستخدم وكلمة المرور كما في Winbox، واضغط
"اتصال". إن ظهر خطأ تسجيل دخول، تحقق من تفعيل **IP → Services → api** في الميكروتيك،
ومن أن المستخدم لديه صلاحية write/full.

إن لم يكن لديك راوتر متاح حالياً، اضغط "معاينة بدون راوتر (بيانات تجريبية)" لرؤية كل
الشاشات ببيانات وهمية كما كانت في النسخة الأولى.

## نظام الترخيص (تجريبي 10 أيام + تفعيل بكود)

التطبيق الآن اسمه **MikroTik-Admin**. أول مرة يُفتح فيها، يبدأ عدّاد تجريبي 10 أيام
محفوظ في ملف على السيرفر نفسه (`backend/data/license.json`) — وليس في متصفح المستخدم،
فلا يمكن تصفيره بحذف بيانات المتصفح. بعد انتهاء المدة، تظهر شاشة تطلب كود تفعيل.

- غيّر كود التفعيل من `backend/config.php` → `ACTIVATION_CODE` (القيمة الحالية مثال فقط).
- غيّر مدة التجربة من نفس الملف → `TRIAL_DAYS`.
- لإعادة التجربة من الصفر يدوياً (مثلاً عند تركيبه على جهاز جديد لعميل آخر)، احذف
  ملف `backend/data/license.json` فقط.
- تأكد أن مجلد `backend/data/` قابل للكتابة (صلاحيات الكتابة) من قبل خادم PHP.

## بنية المشروع

```
frontend/
  index.html        ← الواجهة الكاملة (HTML/CSS/JS في ملف واحد، تعمل بدون أي بناء/Build)
backend/
  config.php         ← اتصال قاعدة البيانات + جلسة الدخول + دالة الاتصال بالراوتر
  classes/
    RouterosAPI.php  ← عميل بروتوكول RouterOS API (PHP خام، بدون مكتبات خارجية)
  api/
    connect.php         → POST تسجيل الدخول (مثل Winbox) — يحفظ الجلسة
    disconnect.php      → POST تسجيل الخروج — يمسح الجلسة
    dashboard.php       → GET  بيانات الموارد والعدادات
    cards_generate.php  → POST توليد كروت (تُنشأ في الميكروتيك + تُؤرشف في قاعدة البيانات)
    cards_list.php      → GET  قائمة الكروت مع فلاتر
    cards_print.php     → POST تعليم كرت كمطبوع
    ping.php             → GET  فحص اتصال عبر أمر /ping في الميكروتيك
    traffic.php          → GET  مراقبة ترافيك واجهة شبكة
    neighbors.php        → GET  الأجهزة المجاورة (MNDP/CDP)
    log.php              → GET  آخر 100 سطر من سجل الميكروتيك
    ippool.php           → GET  مجالات الأيبيهات المستخدَمة/المتاحة
    settings.php         → GET/POST حفظ راوتر افتراضي ثابت في قاعدة البيانات (اختياري)
database/
  schema.sql          ← جميع الجداول + بيانات ابتدائية (تُستخدم فقط لأرشفة الكروت، الاتصال نفسه لا يحتاجها)
```

## خطوات التشغيل (نشر فعلي على استضافة)

1. **قاعدة البيانات**: أنشئ قاعدة MySQL ونفّذ `database/schema.sql` بداخلها. عدّل آخر سطر
   فيه (`routers_config`) بعنوان راوترك الحقيقي ومستخدم/كلمة مرور API.
2. **الباك إند**: عدّل بيانات `DB_HOST` / `DB_USER` / `DB_PASS` في `backend/config.php`،
   ثم ضع مجلد `backend/` على أي استضافة تدعم PHP 8+ (لا حاجة لأي Composer أو مكتبات).
   تأكد من تفعيل **API** في الميكروتيك نفسه: `IP → Services → api` على المنفذ 8728.
3. **الواجهة**: افتح `frontend/index.html` — تعمل حالياً ببيانات تجريبية (Demo) داخل
   `<script>` لتشاهد شكل وحركة كل الشاشات فوراً بدون أي تركيب.
   لربطها بالباك إند الحقيقي: ابحث في الملف عن التعليقات بصيغة **"نقطة ربط:"** وبدّل كل
   دالة عرض بيانات Demo بطلب `fetch()` فعلي إلى نقطة النهاية المقابلة، مثال:
   ```js
   const res = await fetch('/backend/api/dashboard.php');
   const data = await res.json();
   document.getElementById('cpuVal').textContent = data.cpu_load + '%';
   ```
4. **الطباعة الحرارية**: قسم "إعدادات الطباعة" يبني معاينة الكارت بعرض 58mm/80mm.
   لتفعيل الطباعة الفعلية أضف داخل `<head>`:
   ```html
   <style>@media print { @page { size: 58mm auto; margin: 0; } }</style>
   ```
   ثم نفّذ `window.print()` عند الضغط على زر الطباعة (المكان محدد بتعليق في الكود).

## ملاحظات أمان مهمة قبل النشر الفعلي

- شفّر كلمة مرور الميكروتيك في `routers_config` (مثلاً عبر `openssl_encrypt` بمفتاح سري
  بيئي)، الكود الحالي يخزّنها كما هي للتبسيط فقط.
- أضف تسجيل دخول لمن يستخدم لوحة التحكم نفسها (جدول `admin_users` جاهز في `schema.sql`،
  استخدم `password_hash()` و جلسات PHP أو JWT لحمايته).
- استخدم HTTPS دائماً لأن طلبات الـ API تحمل بيانات حساسة (يوزرات/باسوردات الكروت).
- قيّد صلاحيات مستخدم API في الميكروتيك بحسب الحاجة الفعلية (write بدل full إن أمكن).

## إضافات عن التحليل الأصلي

أضفنا جدولين غير مذكورين بالاسم في التحليل لأن الواجهة تتطلبهما عملياً:
`print_templates` (لتخزين اسم الشبكة/الشعار/رقم الدعم/عرض الورق بدل تضمينها كأكواد ثابتة)
و `admin_users` (لحماية الدخول إلى لوحة التحكم ذاتها، بصرف النظر عن بيانات الميكروتيك).
